ACT-BizCard 業務戰情系統 (Frontend)![Tech](https://www.google.com/search?q=https://img.shields.io/badge/Tech-Tailwind_CSS | Cropper.js | GPT--4o--mini-green)這套系統是專為 亞瑞仕國際驗證 (ARES) 等專業服務業設計的「AI 業務戰情系統」。透過手機端採集名片，利用 GPT-4o-mini 進行精準辨識，並實作客戶歸屬追蹤與數據分析。🚀 核心功能AI 名片採集器：支援手機拍照，內建 Cropper.js 進行名片尺寸優化，提升 AI 辨識成功率。GPT-4o-mini OCR：整合 OpenAI 最新模型，將名片圖檔秒轉結構化 JSON 數據。業務資產歸屬：自動鎖定客戶與業務的歸屬關係，防止員工離職導致的客戶流失。V3 戰情發布區：支援所見即所得 (WYSIWYG) 編輯，老闆可輕鬆上架新聞、課程與講座資訊。自動化數據中心：串接 Google Sheets，自動補足電話開頭 0，並凍結標題列便於閱讀。🛠 技術架構前端 (Frontend)：託管於 GitHub，使用 Tailwind CSS 與 JavaScript。邏輯中樞 (Backend)：Google Apps Script (GAS)，負責 Sheets 寫入、Drive 存檔及授權邏輯。AI 橋接層 (API Bridge)：Cloudflare Worker，負責持有 API Key 並執行 GPT-4o-mini 辨識。數據庫 (Database)：Google Workspace (Sheets & Drive)。⚙️ 環境變數設定在使用本系統前，請確保以下環境變數已正確配置：Cloudflare Worker (Secrets)變數名稱說明API_KEYOpenAI API Key (需具備 GPT-4o-mini 權限)GAS_URLGoogle Apps Script 部署後的 Web App 網址Google Apps Script (CONFIG)變數名稱說明WORKER_URL部署後的 Cloudflare Worker 網址LICENSE_KEY開發者遠端授權密鑰 (Kill-Switch)📂 資料夾結構Index.html：業務採集端前端介面。Main_Control.gs：GAS 核心控制邏輯。AI_Service.gs：AI 辨識與資料寫入服務。Worker.js：Cloudflare Worker 原始碼。📦 部署說明Worker：在 Cloudflare 建立新 Worker，貼入 Worker.js 並設定 API_KEY。GAS：在客戶 Google 帳號建立專案，貼入 .gs 檔案並執行 initSystem。Frontend：將 Index.html 內容上傳至本倉庫。若需獨立運行，請確保 GAS 之 doGet 已正確引用。⚠️ 商業授權保護本系統內建「遠端授權驗證機制」。系統每次執行前均會向授權伺服器確認 LICENSE_KEY 狀態。若停止合作，開發者有權從遠端關閉系統執行權限。© 2024 ACT-BizCard System Developer.
